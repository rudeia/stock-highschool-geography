# 교사용 안내

## 무엇을 가르치는가

Market Class는 **중·고등학교 경제 단원의 핵심 개념들을 한 게임 안에서 동시에** 다룹니다.

| 학습 영역 | 게임 내 구현 |
|---|---|
| **분산투자와 위험 관리** | 24종의 자산 (주식·ETF·부동산·선물·채권·외환) 매수·매도 |
| **거시경제 4대 지표의 상호작용** | 기준금리, 부동산지수, 환율, 실업률이 매 라운드 변동 |
| **이슈와 시장 반응의 인과** | 40여 개 경제 이슈 카드가 자산별 가격에 차등 영향 |
| **예금과 적금의 차이** | 보통예금(자유입출금) · 정기예금(만기 잠금) · 정기적금(분할 납입) 비교 카드 |
| **금리·환율·인플레이션의 연쇄효과** | 트리거 시스템 — 임계치 도달 시 정책 자동 발동 |
| **자산 종류별 변동성 차이** | 우량주는 안정적, 중소형주는 변동성 큼 (사이즈 팩터) |
| **분기복리 이자 계산** | 한 라운드 = 1분기, 모든 이자는 분기복리로 누적 |

## 누구를 위한 것인가

- 중·고등학교 **경제·사회 교과** 수업
- **자유학기제·진로 교육** 활동
- **금융 문해력 (Financial Literacy)** 캠프·동아리
- **대학 교양 경제학 입문** 강의의 보조 도구

권장 인원: **6~30명** (한 학급 전체 동시 진행 가능)
권장 시간: **40~50분** (1~2교시)

## 빠른 시작 — 교사용 5분 가이드

### 준비물

- 교사: PC 또는 노트북 (브라우저)
- 학생: 휴대폰 또는 태블릿 (브라우저, 회원가입 불필요)
- 교실 와이파이 또는 핫스팟

### 진행 순서

1. **방 만들기**
   - 교사 화면에서 "새 방 만들기" → 학급 이름 입력
   - 6자리 PIN 코드 생성됨

2. **학생 접속**
   - 학생들에게 사이트 주소와 PIN 공유
   - 학생들은 닉네임만 입력하고 즉시 참여

3. **라운드 진행 (반복)**
   - 교사가 "라운드 시작" → 이슈 카드 3~5개 공개
   - 학생들이 60~90초 동안 매수·매도 결정
   - 교사가 "라운드 종료" → 가격 자동 갱신, 거시지표 변동

4. **거시지표 모니터링**
   - 교사 대시보드에서 금리·환율·부동산·실업률 실시간 확인
   - 임계점 돌파 시 트리거 이벤트 자동 발동 (예: 실업률 8% 돌파 → 긴급 경기부양 패키지)

5. **게임 종료 후 토론** (15분)
   - 수익률 상위·하위 학생의 포트폴리오 비교
   - "왜 같은 이슈에서 누구는 벌고 누구는 잃었나?" 토론
   - 거시지표 그래프 보며 인과 추적

## 게임 메커니즘

### 자산 24종

| 분류 | 종목 | 개수 |
|---|---|---|
| 우량주 | 코어실리콘, 한양반도체, 한끼정유, 글로벌오일 외 | 5 |
| 중소형주 | 네오모빌리티, 스카이항공, 오션항공, 에코에너지 외 | 10 |
| ETF | 코스피200, S&P500 | 2 |
| 부동산 ETF | 도시부동산지수 추종 ETF | 1 |
| 선물 | 원유 선물, 곡물 선물, 금 선물 | 3 |
| 채권 | 미국채, 아르헨티나 채권 (액면가 100,000원) | 2 |
| 외환 | 원/달러 환율 ETN | 1 |

### 이슈 시스템

- **일반 이슈 30+종** — 호황, 인플레이션, 고용, 정책, 글로벌 이벤트 등
- **트리거 이슈 6종** — 거시지표가 임계점 도달 시 자동 발동
  - 실업률 > 8.0% → 긴급 경기부양 패키지
  - 실업률 < 2.5% → 임금·물가 악순환
  - 기준금리 > 7.0% → 신용경색
  - 기준금리 < 1.0% → 유동성 과잉
  - 환율 > 1,600원 → 외환당국 시장 개입
  - 부동산지수 > 임계점 → 부동산 규제 패키지

### 거시 ↔ 자산 양방향 효과

이슈가 자산 가격에만 영향을 주는 것이 아니라, **거시지표도 함께 변동**합니다. 그리고 변동된 거시지표가 다음 라운드의 트리거를 발동시키는 **연쇄 효과**가 발생합니다.

```
[5R] 호황 이슈 → 부동산 지수 상승
[6R] 부동산 지수 임계점 돌파 → 규제 패키지 자동 발동
[7R] 규제로 부동산 자산 동반 하락 + 금리 인상
[8R] 금리 인상으로 채권 가격 하락, 신흥국 채권 직격
```

### 예금·적금·정기예금 비교

- **보통예금** (자유입출금): 언제든 입출금, 분기복리 이자
- **정기예금**: 목돈 한 번에 예치, 4라운드(=1년) 잠금, 우대금리 +0.6%p
- **정기적금**: 매 라운드 분할 납입 (비교 학습 카드로 시각화)

정기예금과 정기적금의 **이자 차이를 막대 그래프로 비교**하는 학습 카드가 학생 화면에 자동 표시됩니다.

## 학습 시나리오 예시

### 시나리오 1 — "인플레이션과 금리 인상 사이클" (40분)

- **목표**: 인플레이션 → 금리 인상 → 채권 가격 하락 → 안전자산 선호의 흐름 이해
- **진행**: 인플레이션 관련 이슈를 의도적으로 자주 노출 (8라운드)
- **토론 포인트**: "왜 금리가 오르면 채권 가격이 떨어지나?"

### 시나리오 2 — "외환위기 대응" (45분)

- **목표**: 환율 급등 시 수출주·수입주의 차이, 외환보유고의 역할
- **진행**: 환율 1,600원 트리거 발동까지 유도, 외환당국 개입 이벤트 학습
- **토론 포인트**: "외환 개입이 단기와 장기에 어떻게 다른가?"

### 시나리오 3 — "분산투자의 가치" (50분)

- **목표**: 한 섹터 집중투자 vs 분산투자의 결과 비교
- **진행**: 학생을 두 그룹으로 나눠 한쪽은 항공주만, 다른 쪽은 분산투자
- **토론 포인트**: 위기 이슈 발생 시 두 그룹 수익률 비교

# 개발자 가이드

## 기술 스택

| 영역 | 기술 |
|---|---|
| Frontend | React 18, Vite, JSX |
| 상태 관리 | React Hooks + Context |
| Backend | Supabase (PostgreSQL + Realtime) |
| 인증 | Anonymous Auth (학생용 닉네임 기반) |
| 배포 | Vercel |
| 스타일링 | CSS (single `styles.css`) |

## 설치

### 필수 요구사항

- Node.js 18+
- npm 또는 yarn
- Supabase 계정 (무료 플랜으로 충분)

### 단계별 설치

```bash
# 1. 저장소 클론
git clone https://github.com/your-org/market-class.git
cd market-class

# 2. 의존성 설치
npm install

# 3. Supabase 프로젝트 생성
#    https://supabase.com/dashboard 에서 새 프로젝트 생성
#    프로젝트 URL과 anon key 확인

# 4. 환경 변수 설정
cp .env.example .env
# .env 파일을 열어서 Supabase 정보 입력

# 5. 데이터베이스 마이그레이션 실행
#    Supabase 대시보드 → SQL Editor에서 순서대로 실행:
#    1) supabase/migrations/20260615000000_initial_market_class_schema.sql
#    2) supabase/migrations/20260616120000_savings_forex_triggers.sql

# 6. 로컬 개발 서버 실행
npm run dev

# 7. 프로덕션 빌드
npm run build

# 8. Vercel 배포
vercel --prod
```

## 데이터베이스

### 주요 테이블

| 테이블 | 용도 |
|---|---|
| `rooms` | 게임 방 (PIN, 현재 라운드, 거시지표, 트리거 쿨다운) |
| `players` | 참여 학생 (현금, 예금, 정기예금, 보유 자산) |
| `student_states` | 학생별 라운드 스냅샷 |
| `team_accounts` | 팀 모드용 계좌 |
| `assets` | 자산 마스터 (24종) |
| `issues` | 이슈 카드 마스터 |
| `round_events` | 라운드별 발동된 이슈 기록 |

### 마이그레이션 실행 순서

1. `20260615000000_initial_market_class_schema.sql` — 초기 스키마 (테이블 생성)
2. `20260616120000_savings_forex_triggers.sql` — 외환 자산·정기예금·트리거 쿨다운 컬럼 추가

두 마이그레이션 모두 **idempotent** (여러 번 실행해도 안전).

### Row Level Security (RLS)

- 모든 테이블에 RLS 활성화
- 학생은 자신이 속한 방의 데이터만 읽기/쓰기 가능
- 교사는 자신이 생성한 방 전체 제어 가능

## 게임 로직 핵심 함수 (App.jsx)

| 함수 | 역할 |
|---|---|
| `resolveRound()` | 라운드 결정 — 이슈 적용, 가격 갱신, 거시지표 변동 |
| `applySizeFactor()` | 자산 크기별 이슈 영향 차등화 |
| `getPassiveMarketMove()` | 이슈 외 노이즈 (게이미피케이션) |
| `detectMacroTriggers()` | 거시지표 임계점 검사 → 자동 발동 이벤트 |
| `accrueTimeDepositInterest()` | 정기예금 분기복리 이자 누적 |
| `maybeMatureTimeDeposit()` | 만기 도래 시 자동 환입 |
| `simulateSavingsComparison()` | 정기예금 vs 정기적금 비교 시뮬레이션 |
| `getQuarterLabel()` | 라운드 → 분기 표기 변환 (한 라운드 = 1분기) |
