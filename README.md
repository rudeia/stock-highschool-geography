## 프로젝트 개요

`Market Class`는 Kahoot!처럼 교사가 방을 만들고, 학생이 QR 코드로 참여하는 경제 교육용 웹 애플리케이션입니다.

학생은 1억 원의 초기 자본과 라운드별 300만 원의 투자 가능 소득을 받습니다. 교사가 선택한 이슈를 보고 주식, ETF, 부동산, 선물, 외환, 채권, 예금에 자산을 배분하며, 장 마감 후 가격 변화와 경제 원리 해설을 확인합니다.

실제 시세 API를 사용하지 않고 앱에 내장된 가상 기업·자산·이슈 데이터로 진행되므로, 수업 목표에 맞게 반복 실험할 수 있습니다.

## 핵심 기능

### 교사

- 사전 등록된 교사 ID별 독립 방 운영
- 6자리 PIN과 학생 접속용 QR 코드 자동 생성
- 4라운드(1년형) 또는 12라운드(3년형) 선택
- 개인 투자 또는 모둠 공동 투자 모드 선택
- 라운드 시작, 장 마감, 일시 정지, 다음 라운드, 게임 종료 제어
- 라운드별 최대 5개 이슈 선택, 초기화, 랜덤 이슈 3개 선택
- 거시 지표, 전체 자산 시황, 학생 접속 상태, 순위 및 포트폴리오 모니터링
- 라운드별 이슈 결과, 거시 트리거, 상장폐지 결과 해설
- 미제출 학생 보고서 일괄 마감, 인쇄, CSV 다운로드

### 학생

- QR 코드로 접속한 뒤 학급 번호, 이름, 6자리 개인 비밀번호로 참여
- 현금, 예금, 투자 자산, 명목·실질 수익률 대시보드
- 종목별 가격 이력, 기업 스토리, 간이 재무제표, 위험 태그, 투자 체크리스트
- ETF·부동산·선물·외환·채권의 상품 구조, 수익 원천, 핵심 위험, 게임의 단순화 규칙
- 장 진행 중 매수·매도, 전량·절반·금액 지정 거래
- 장 마감 후에도 가능한 예금 입·해지로 유동성 비교
- 라운드 메모, 학습 체크포인트, 거래 로그, 이슈 분석, 최종 자기평가
- 최종 보고서 제출 및 브라우저 인쇄 기능을 이용한 PDF 저장
- 재접속 시 Supabase에 저장된 잔고, 보유 종목, 로그, 메모 복원

## 수업 규칙

| 항목 | 규칙 |
|---|---|
| 초기 자본 | 게임 시작 시 개인 또는 모둠 계좌에 1억 원 지급 |
| 라운드 소득 | 매 라운드 300만 원 지급. 기본 생활비를 제외한 분기별 투자 가능 소득으로 가정 |
| 시간 | 1라운드 = 1분기. 4라운드는 1년, 12라운드는 3년 |
| 접속 인원 | 방당 최대 40명 |
| 방 유지 | 생성 후 24시간. 만료된 방은 조회 대상에서 제외되고 정리 함수의 대상이 됨 |
| 거래 시간 | 주식·ETF·선물·외환·채권은 장 진행 중에만 매수·매도 |
| 거래 비용 | 매수·매도 수수료 0.25%, 매도 거래세 0.18% |
| 이슈 | 라운드별 최대 5개. 성공 확률 70%, 성공 경로 중 30%는 사건 확정 대신 기대감만으로 가격에 반영 |
| 배당 | 3·6·9·11라운드 마감 보유량 기준. 성장 0%, 안정 5%, 고배당 10%에 방별 ±1%p 난수 |
| 배당락 | 1주당 배당금의 50%만큼 주가에 반영 |
| 상장폐지 | 9라운드 이후, 기업에 직접 연결된 악재 2개 이상이 2라운드 지속될 때 20% 확률 |
| 자연 변동 | 이슈가 없어도 라운드별 시장 노이즈 반영. 자산 특성별 변동성 차등 |

## 투자 자산

총 25개의 가상 자산을 사용합니다. 방을 만들 때 각 자산의 초기 가격 후보 3개 중 하나가 랜덤 배정됩니다.

| 분류 | 수 | 자산 |
|---|---:|---|
| 국내·해외 주식 | 16 | 네오모빌리티, 코어실리콘, 에코에너지, 글로벌오일, 유니버스엔터, 미래푸드, 스카이항공, 대한은행, 메디케어, 한빛인프라, 도지인마스, 한강은행, 오션항공, 바른푸드, 메트로인프라, 제노믹스바이오 |
| 지수 ETF | 2 | S&P 500 ETF, KOSPI 200 ETF |
| 부동산 ETF | 1 | 도시부동산지수 추종 ETF |
| 원자재 선물 | 3 | 글로벌 원유 선물, 글로벌 곡물 선물, 글로벌 금 선물 |
| 외환 | 1 | 원/달러 환율 추종 ETN |
| 채권 | 2 | 미국 10년 국채, 아르헨티나 국채 |

기업의 재무제표는 `재무 안정형`, `성장 투자형`, `고부채 확장형` 중 하나로 방별 랜덤 배정됩니다. 금리, 환율, 원자재, 고용, 규제 이슈에 대한 반응 강도가 부채비율·현금·수출 비중·연구개발 비중 등에 따라 달라집니다.

## 거시경제와 이슈 모델

### 거시 지표

- 기준금리
- 예금금리
- 부동산지수
- 원/달러 환율
- 실업률
- 물가지수와 누적 수요견인 인플레이션

방 생성 시 초기 금리·실업률·환율, 이슈 강도, 트리거 민감도, 인플레이션 민감도가 난수로 정해집니다. 따라서 같은 이슈를 선택해도 방마다 시장의 반응이 완전히 같지 않습니다.

### 이슈 범주

- 경기·고용·물가
- 금리·환율
- 원자재·식량
- 전쟁·정치
- 채권·국가 신용
- 기술·산업
- 부동산·소비

상충하는 이슈가 동시에 선택되면 동일 경향성의 결과가 무조건 합산되지 않도록 조정됩니다. 또한 거시 지표가 임계치를 넘으면 긴급 경기부양, 임금·물가 악순환, 신용경색, 유동성 과잉, 외환시장 개입, 부동산 규제 같은 자동 트리거가 발동합니다.

## 배당·예금·채권 학습

- 주식은 방 생성 시 성장주, 안정 배당주, 고배당주 중 하나의 배당 성향을 받습니다.
- 배당 지급과 배당락이 함께 반영되어 `배당 = 공짜 수익`이 아님을 학습합니다.
- 보통예금은 자유롭게 입·해지하며 분기별 복리 이자를 받습니다.
- 정기예금은 4라운드 잠금과 우대금리를 적용하고, 중도 해지 시 페널티가 있습니다.
- 채권은 가격 변화와 액면가 기준 이자를 함께 반영하여 금리 위험과 신용 위험을 비교합니다.

## 모둠 투자

모둠 모드에서는 최대 8개 모둠이 각각 1억 원의 공동 계좌를 사용합니다.

- 거래권을 활성화한 모둠원만 공동 계좌로 거래할 수 있습니다.
- 거래권에는 시간 제한이 있어 여러 학생의 동시 거래 충돌을 줄입니다.
- 포트폴리오와 현금·예금은 모둠이 공유하고, 학습 메모와 최종 회고는 개인별로 제출합니다.
- 공동 계좌의 음수 상태가 2라운드 지속되면 파산 규칙이 적용됩니다.

## 최종 보고서

게임이 끝나면 학생은 다음 내용을 확인하고 제출할 수 있습니다.

- 최종 자산, 현금성 자산, 투자 자산, 수익률, 실질 구매력
- 자산별 보유 비중과 거래 로그
- 라운드별 이슈 성공·기대 반영·실패 결과와 관련 자산
- 거시 지표 변화와 자동 트리거
- 라운드 메모와 체크포인트 답변
- 잘한 점, 부족한 점, 다음 투자 계획
- 포트폴리오 성향에 따른 투자자 유형

교사는 제출 현황을 확인하고, 재접속하지 못한 학생이 있으면 `제출 마감`으로 현재 저장 상태를 자동 제출할 수 있습니다.

## 권장 수업 흐름

1. 교사가 로그인하고 새 수업 방을 만듭니다.
2. 라운드 수, 개인·모둠 모드, 시장 변동성을 선택합니다.
3. 학생이 QR 코드로 접속하고 교사가 접속 현황을 확인합니다.
4. `게임 시작`을 눌러 초기 자본을 지급합니다.
5. 교사가 이슈를 고르거나 `랜덤 이슈 3개로 장 시작`을 선택합니다.
6. 학생은 이슈의 결과를 추론하며 자산을 매매하고 라운드 메모를 작성합니다.
7. 교사가 장을 마감하면 이슈 판정, 가격, 배당·이자, 거시 지표, 순위가 갱신됩니다.
8. 라운드 해설과 포트폴리오 변화를 함께 토론합니다.
9. 최종 라운드 후 학생이 자기평가를 완성하고 보고서를 제출합니다.
10. 교사가 미제출자를 마감하고 CSV를 다운로드한 뒤 게임을 종료합니다.

## 로컬 실행

### 1. 요구 환경

- Node.js 20 이상 권장
- npm
- 실시간 다중 기기 수업을 위한 Supabase 프로젝트

### 2. 설치

```bash
git clone <YOUR_REPOSITORY_URL>
cd <YOUR_REPOSITORY_DIRECTORY>
npm install
```

### 3. 환경변수

`.env.example`을 `.env`로 복사하고 Supabase 프로젝트 값을 입력합니다.

```bash
cp .env.example .env
```

```env
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-supabase-publishable-key
```

기존 Supabase 프로젝트의 anon key를 사용하는 경우에도 대응합니다.

```env
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
```

> `.env`는 GitHub에 올리지 마세요. `.gitignore`에서 제외하고 있습니다.

### 4. Supabase 데이터베이스

Supabase Dashboard의 `SQL Editor`에서 아래 파일을 순서대로 실행합니다.

1. `supabase/migrations/20260615000000_initial_market_class_schema.sql`
2. `supabase/migrations/20260616120000_savings_forex_triggers.sql`

첫 번째 파일은 방, 학생, 모둠 계좌, 자산, 이슈, 라운드 결과, 학생 상태, 최종 제출 테이블과 Realtime 설정을 만듭니다. 두 번째 파일은 외환 자산, 정기예금, 거시 트리거 필드를 보강합니다.

### 5. 개발 서버

```bash
npm run dev
```

기본 주소는 `http://localhost:5173` 입니다.

### 6. 검사와 빌드

```bash
npm run lint
npm run build
npm run preview
```

`npm run build`가 완료되면 배포용 결과물이 `dist/`에 생성됩니다. `dist/`는 소스 파일이 아니므로 GitHub에 올리지 않아도 됩니다.

## Vercel 배포

1. GitHub에 이 프로젝트를 올립니다.
2. Vercel에서 `Add New` → `Project`를 선택하고 저장소를 가져옵니다.
3. Framework Preset을 `Vite`로 선택합니다.
4. 아래 빌드 설정을 확인합니다.

```text
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

5. Vercel `Project Settings` → `Environment Variables`에 다음 값을 추가합니다.

```text
VITE_SUPABASE_URL
VITE_SUPABASE_PUBLISHABLE_KEY
```

6. `Production`, `Preview`, `Development`에 필요한 범위를 선택하고 재배포합니다.

`vercel.json`에 Vite 빌드 설정과 SPA 라우팅 rewrite가 포함되어 있습니다.

## 교육적 범위와 한계

이 앱은 경제 원리의 인과관계를 토론하기 위한 **교육용 단순화 모델**입니다.

- 실제 주가·금리·환율 API와 연결되지 않습니다.
- 이슈와 자산의 반응은 교육용 가중치와 확률에 따라 계산됩니다.
- 선물의 레버리지·만기·롤오버, ETF·ETN의 보수·추적오차·발행사 위험, 채권의 정교한 듀레이션은 일부 생략되어 있습니다.
- 특정 이슈가 항상 같은 결과를 만들지 않도록 확률과 방별 난수를 사용합니다.
- 실제 투자 판단이나 수익률 예측에 사용하면 안 됩니다.

교사는 라운드 해설에서 `왜 그런 반응이 나왔는가?`, `반대 결과가 나올 조건은 무엇인가?`를 함께 토론하는 방식으로 사용하는 것을 권장합니다.

## 보안 주의사항

현재 교사 로그인은 프론트엔드에 사전 등록된 ID를 확인하는 **수업용 간이 잠금**입니다. Supabase 정책 또한 프로토타입 수업 운영을 위해 넓은 읽기·쓰기를 허용합니다.

따라서 이 상태로는 학급 내 제한된 교육용 운영에 적합하며, 공개 서비스·실명 정보·중요 데이터를 다루는 운영에는 적합하지 않습니다. 공개 서비스로 확장할 때는 다음 작업이 필요합니다.

- Supabase Auth 기반 교사 계정과 세션
- 교사·학생·방 소유권을 반영한 RLS 정책
- 서버 측 권한 검증
- 로그 보존, 개인정보 최소화, 관리자 기능
